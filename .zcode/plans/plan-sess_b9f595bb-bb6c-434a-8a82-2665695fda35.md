## Parallel OCR Batch Processing

### Problem
23 pages → 3 batches of 8 images → 3 **sequential** Gemini API calls (~2 min each) = ~6 min total. Nginx times out at 5 min.

### Solution
Run all Gemini API calls in **parallel** using `asyncio.gather()` + the Gemini SDK's native async client (`client.aio`).

**Before**: Batch 1 → wait → Batch 2 → wait → Batch 3 → done (~6 min)
**After**: Batch 1 ┐
               ├→ done (~2 min)
               ┘

### Changes

**1. `backend/app/ocr/gemini.py`**
- Add `async def _process_batch_async()` using `client.aio.models.generate_content()`
- Add `async def extract_rows_from_images_async()` using `asyncio.gather()` for concurrent batches
- Keep existing sync function for backward compatibility

**2. `backend/app/routes/ocr.py`**
- Change: `rows = extract_rows_from_images(batches)` → `rows = await extract_rows_from_images_async(batches)`

### Expected Result
- 3 parallel Gemini calls instead of 3 sequential
- Total OCR time: ~2 min (limited by slowest batch only)
- No nginx timeout issues
